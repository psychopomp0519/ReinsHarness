# Session Wrap

Wrap up the current session by collecting observations, generating action items, and saving follow-ups for the next session.

## Phase 0: Context Collection

1. Run `git diff --stat` to see what changed in this session.
2. Gather session observations: files modified, key decisions made, patterns discovered.

## Phase 1: Generate Action Items

Analyze the session and generate action items in three categories:

- **[docs]** — Documentation updates needed (README changes, inline comment improvements, API docs)
- **[learning]** — Learnings to capture (patterns discovered, pitfalls encountered, useful techniques)
- **[followup]** — Follow-up tasks (TODOs, pending fixes, next steps, deferred work)

## Phase 2: Present Categorized List

Present action items as a numbered list with category tags:

```
Session Wrap-Up
───────────────
1. [docs]     Update README with new CLI flags
2. [docs]     Add JSDoc to exported functions in utils.ts
3. [learning] Discovered that X pattern works better than Y
4. [followup] Refactor auth module to use new token flow
5. [followup] Add integration tests for DB guard hook
```

## Phase 3: User Selection

Ask the user which items to execute:

- Enter numbers (e.g., `1, 3, 5`)
- Enter `all` to execute everything
- Enter `none` to skip execution (still saves follow-ups)

## Phase 4: Execute Selected Items

Execute the selected doc and learning items immediately. For follow-up tasks, save them.

After execution, save all remaining follow-up tasks to `.reins/session-followups.md` with this format:

```markdown
# Session Follow-ups
<!-- Auto-generated by reins-session-wrap -->
<!-- Date: YYYY-MM-DD -->

## Pending
- [ ] Task description (from session wrap on YYYY-MM-DD)

## Completed
- [x] Completed task description
```

Append to existing file if it already exists — do not overwrite previous follow-ups.
